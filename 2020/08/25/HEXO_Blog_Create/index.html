HEXO Blog Create

2020-08-25 Hexo Hexo, Hexo_Blog, Hexo_Command Comments Word Count: 258(words) Read Count: 1(minutes)

HEXO 셋업에서 가장 중요한 포인트만 정리하였습니다.

1.Git Setup

클릭 하여 다운로드 페이지로


2.Node Setup

클릭 하여 다운로드 페이지로 이동 합니다.
이후 가장 최신의 LTS 버전을 다운로드 합니다.


3.Create empty Repository

Repository 는 관리용과 게시용 두개로 나누어 사용합니다.
관리용 Repository에는 Hexo의 모든 구성요소가 저장되고
게시용 저장소에서는 Hexo에서 빌드된 뷰파일만 업로드 됩니다.

관리용 저장소의 생성
Repository 이름 github유저이름.Blog

게시용 저장소의 생성
Repository 이름 github유저이름.github.io



4.init Hexo

Hexo의 파일구조와 Config문서 여기를 참고해 주세요



5.One-Command Deployment

hexo-deployer 플러그인을 이용하여 설정된 다양한 저장소에 hexo를 deploy하여 배포할 수 있습니다


Hexo의 루트 디렉토리에 존재하는 config 하단부의 deploy 설정을 아래의 양식에 맞추어 변경합니다.
repositoryURL은 이곳에서 생성한 게시용Repository로 설정해 주세요.


위 명령어를 통해 hexo를 deploy하여 배포할 수 있습니다


6.Save All Hexo source

관리용 Repository에 Hexo를 운영하는 폴더 전체를 커밋하시면
이후에 다른곳에서도 게시글을 작성하고 Deploy가 가능합니다.
