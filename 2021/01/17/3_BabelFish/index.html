3_babelFish 프로젝트 - DB 구성

2021-01-17 MySQL BabelFish, Project Comments Word Count: 352(words) Read Count: 2(minutes)

Mysql

MYSQL은 1995년에 발표된 오픈 소스 관계형 DBMS입니다
MYSQL을 선택한 이유는 학교에서 배웠던 데이터 구조 설계를 복습하며 동시에 실 프로젝트에서 적용해보기 위해
이번 프로젝트 DBMS를 MYSQL로 선택하게 되었습니다.



요구사항 분석

이번 프로젝트에서 DBMS에 질의를 거쳐야 하는 기능들의 리스트를 정의하였습니다

  • 유저 등록 (비밀번호는 암호화해서 저장되어야 합니다)
  • 유저 비밀번호 찾기
  • 유저 프로필 수정
  • 유저별 단어장 등록
  • 단어장 정보 수정
  • 단어장 단어 추가
  • 단어장 단어 삭제
  • 단어장 단어 수정
  • 단어시험 중 오답일 경우 오답 횟수 저장
  • 단어시험 중 만점일 경우 학습 완료로 간주하고 학습 완료 일자, 학습 카운트 기록



ER 다이어그램 작성

분석된 요구사항을 바탕으로 ER다이어그램을 작성하였습니다



함수종속 다이어그램 표현


이후 ER 다이어그램을 토대로 정규화를 위해 함수종속 다이어그램을 작성했습니다



정규화


애자일 개발방법론을 지향하기에 개발 중 필요에 따라 릴레이션의 속성이 추가되거나 삭제될 수 있는 가능성이 높을 수밖에 없어요
관계형 데이터베이스에서는 이럴 경우 이상 현상이나 불필요한 데이터가 발생할 수 있기 때문에
BCNF까지의 정규화 과정을 통해 위험부담을 최소화하고자 했습니다



릴레이션 스키마 구성

정규화된 다이어그램을 참조해서 릴레이션 스키마를 구성했습니다



쿼리 작성

구성한 스키마를 토대로 테이블을 구성하였습니다.
