5_babelFish 프로젝트 - React를 이용한 프론트엔드 구성기

2021-02-03 React BabelFish, Project, REST API, React Comments Word Count: 322(words) Read Count: 2(minutes)

1. 리액트를 처음 사용하며

이런 프로젝트를 처음 할때는 바닐라자바스크립트 HTTP CSS만 사용해서 만들어보는것이 좋다는 조언도 많이 들었습니다
무엇보다 대학교 수업이나 혼자서 독학으로 꾸준히 사용해봤던것이기 때문에
프론트엔드를 구성하는데 막히는 부분 없이 할 수 있다는 장점도 분명 있다고 생각합니다

하지만 잘 몰라도 열심히 검색하며 백엔드 서버를 구성해 자신감도 어느정도 생겼고!
새로고침이 없이 자연스러운 웹앱으로 멋있게 만들어보고 싶어 배워보자는 마인드로 리액트를 선택했습니다

2.요구사항 분석

걱정은 그다음날의 자신에게 맡긴뒤 제가 이 프로젝트에 원하는 요구사항을 마음껏 정해보았습니다

(🌟 한번도 해본적이 없어 난관이 예상되는 도전적인 기능)
🌟 단어를 학습하기에 좋은 깔끔한 대시보드형태의 레이아웃
🌟 모바일에서도 앱처럼 사용할 수 있는 반응형 레이아웃
🌟 애플 사용자 인터페이스 개발자정보처럼 탄탄한 인터페이스 디자인을 구현
신뢰성 높은 파란색 디자인 색깔이 브랜드의 이미지를 좌우한다? / YTN 사이언스

  • 로그인
  • 단어장별 복습 플랜 추천
  • 단어장 리스트 (추가,수정,삭제)
  • 단어 리스트 (추가, 수정, 삭제)
  • 🌟 단어 시험

3. 초기 구성

처음 리액트를 시작하는 상태였기 때문에 공식문서를 참고해 CRA(Create-React-App) 툴체인으로 쉽게 리액트 환경을 구축했습니다


그리고 앱의 구조도 아래의 사이트를 참고하여 구성했습니다

containers -> 최상위 컴포넌트 유저 데이터 관리 및 하위 컴포넌트 호출
components -> 최상위 컴포넌트에서 호출하는 형태로 사용 모든 데이터는 containers 받아서 사용
res -> 컴포넌트에서 사용되는 이미지,svg 리소스 폴더
serive -> API 처리, 시험문제 생성 모듈등 각종 외부 모듈 폴더

React 프로젝트 구조를 설계하기 위해 추천할 만한 방법
Component 생성 및 모듈화
